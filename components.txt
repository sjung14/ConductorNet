System Pipeline
1. Motion Source (GUI -> Wand) 
2. Motion State
3. Motion -> Control Mapper 
4. Audio Engine

=================================================

Host:
1. Find audio file with filename in directory
2. Make numpy array from audio file
3. Call back function
    a. Modify Numpy array with info over wifi (from wand)
    b. Play short interval of audio (smaller than hardware frames, ideally IMU sample rate)
    c. when song ends, stop call back

Wand:
1. Read IMU short intervals
2. Process raw IMU
3. Send packets over wifi

=================================================

Controls
1. Motion energy (ax^2 + ax^2 + az^2) -> Tempo
    speed of music play (have GUI draw circles so player knows x1.0 speed)
2. Gyro magnitude (gx^2 + gy^2 + gz^2) -> Responsiveness
    low -> more smoothing, high -> little smoothing
3. Wrist twist (ax) ... atan2(ay, az)? -> Volume
    tilted right -> louder volume, tilted left, smaller volume
4. Pointing up/down (az) ... atan2(ax, az)? -> Pitch
    pointing up -> high pass filter, pointing down -> low pass filter

Motion Source -> Motion State -> Motion2Control Mapper -> Audio Engine
